# Phase 6 実装計画: データ移行・デプロイ・最終調整

## 概要
開発した「収支管理ウェブアプリ」に既存のスプレッドシートデータをインポートし、Vercel へのデプロイを完了させて本番利用可能な状態にします。また、ビルドエラー等の修正を含む最終的な品質調整を行います。

---

## 1. データ移行 (Data Migration)
既存の Google スプレッドシートからエクスポートした CSV データを、システムの DB (Supabase) に一貫性を保ったままインポートします。

### 実装・実行内容
- **[NEW]** `scripts/import_spreadsheet_data.ts` (または同等のスクリプト)
  - CSV ファイルを読み込み、以下の順序でデータを変換・登録します：
    1. カテゴリ・口座の名前を ID に変換（マスタにない場合は新規作成またはエラー報告）。
    2. 取引データ（transactions）の登録。
    3. 固定費（auto_payments）、減価償却（depreciation）データの登録。
  - 重複登録を防ぐための簡易的なチェック機能を実装します。

---

## 2. デプロイと最終調整 (Deployment & Polish)
Vercel でのビルドおよびデプロイを成功させ、本番環境特有の設定を行います。

### 実装・実行内容
- **ビルドエラーの解消**
  - Recharts や Zod の型定義と TypeScript バージョンの不整合に伴うエラーを修正済み。
- **環境変数の最終確認**
  - `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - `SUPABASE_SERVICE_ROLE_KEY` (クイック入力 API 用)
- **ミドルウェアの設定**
  - `/api/` ルートが API キー認証を正しく受け付け、セッション強制リダイレクトが発生しないことを確認。

---

## 3. 検証計画
- **インポート検証**
  - サンプルの CSV データをインポートし、ダッシュボードや分析ページに正しく反映されるか確認。
- **デプロイ検証**
  - Vercel のプレビュー URL でアプリケーションが正常に動作し、ログイン・取引登録が可能か確認。
- **API 検証**
  - 本番 URL に対して API キーを使用した `POST /api/transactions` を実行し、成功することを確認。

---

## ユーザーレビューが必要な項目
- **スプレッドシートのデータ形式**: CSV に書き出す際の列順やヘッダー名がスクリプトと一致している必要があります。
- **APIキーの再発行**: 本番環境に移行後、APIキーを再発行して iOS ショートカット等の設定を更新する必要があります。
